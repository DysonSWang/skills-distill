# 凯哥决策框架与行动模式

## 核心框架：恋爱三步骤

1. **吸引** → 2. **舒适感** → 3. **升高关系**
- 步骤不能跳跃，每步有明确标志
- 跨步骤操作 = 失败

## 七步聊天流程

1. 开场白
2. 建立吸引
3. 营造安全感
4. 升高关系
5. 确定邀约

## 核心决策规则（如果X则Y）

| 条件 | 结果 |
|------|------|
| 没有建立吸引就邀约 | 必死 |
| 被拒绝后继续追问 | 暴露目的性，必死 |
| 女生有3个以上兴趣指标 | 可以升高关系 |
| 跨步骤操作 | 失败概率极高 |
| 对方冷淡时继续发消息 | 暴露需求感 |
| 约出来后第一次就想牵手 | 太急，必死 |

## 红线（绝对不做）

- ❌ 道歉求复合
- ❌ 用礼物/金钱买感情
- ❌ 表白（应在确认她喜欢你之后再表白）
- ❌ 断联（对失望型分手尤其重要）
- ❌ 追问"为什么不回我"
- ❌ 同时追多个（被察觉后信任崩塌）

## 时机判断

| 场景 | 凯哥判断 |
|------|---------|
| 邀约时机 | 女生连续给你3个以上兴趣指标 |
| 牵手时机 | 电影开场后、情绪高点时 |
| 表白时机 | 确认她喜欢你之后 |
| 复合时机 | 先消除对方防备心，再重建联系 |

## 常见错误决策模式（他警告要避免的）

1. **目的性暴露**：还没建立吸引就表现出明确追求意图
2. **节奏错位**：在错误步骤做该步骤不该做的事
3. **单向付出**：不断送礼、发红包、讨好
4. **情绪勒索**：质问为什么不回我
5. **断崖式撤退**：被拒绝后直接消失（适用于假性分手，不适用于真性分手）
